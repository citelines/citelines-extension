Design Notes — Light Mode

  • Detection: YouTube uses class dark on html for dark theme, absent for light. Use this to toggle our styles via html:not(.dark) .citelines-* selectors
  • Video player stays dark: YouTube's player is always dark regardless of theme, so the floating buttons (+, bibliography, account) don't change
  • Timeline background: White (#fff) with light grey borders (#ddd/#eee) instead of dark grey (#181818/#2a2a2a)
  • Creator orange: Shifted from #ffaa3e to #e8941a (darker, richer) for sufficient contrast on white background
  • Teal unchanged: #0497a6 works on both light and dark backgrounds — no adjustment needed
  • Others grey: Lifted from #555 to #b0b0b0 — needs to be lighter to read as "muted" on white rather than "bold"
  • Tooltips & popups: White background, light border, subtle shadow — inverted from current dark panel style
  • Playhead: Dark (#1a1a1a) at 25% opacity instead of white at 40% opacity
  • Focus rings: Teal outline with subtle teal shadow instead of just border-color change — more visible on white
  • Badge colors in popups: White text on colored background instead of dark text — better contrast on light popup
  • Implementation approach: CSS-only via html:not(.dark) overrides — no JS theme detection needed. All light-mode rules override the existing dark-mode defaults
  • Bookmark markers: SVG bookmark shapes with hash fill work on both themes — the teal (#0497a6) has enough contrast on white. Hash pattern stroke opacity adjusted slightly lighter for light mode
  • Bookmark lane: Dashed teal separator and tinted background adapt naturally — rgba values look correct on both white and dark backgrounds
  • Scope: Only the Citelines UI adapts — we don't control YouTube's own elements
